### Context
- Component: QBO → Bronze → Auth subsystem
- Focus area: token rotation + serialization contracts

### Goal
- Safe, atomic JSON read/write for:
  - auth tokens (`token.json`)
  - API responses (tables)
- Enforce separation:
  - auth rotation ⟂ ingestion (Spark)
  - workspace/credential mapping ⟂ auth core logic


### Decisions / Invariants

#### Serialization
- All JSON read/write uses **bytes (`.encode("utf-8")`)**
- Writes go through `atomic_write_bytes(...)`
- Reason: prevent partial writes / corruption

#### Auth System Separation
- Auth rotation is **control-plane**, not part of distributed ingestion
- Spark workers:
  - read-only auth snapshot
  - no mutation allowed

#### Execution Modes
- Spark:
  - preflight batch rotation
- Pandas:
  - per-entity refresh (on-demand)

#### Token Expiry
- Token lifetime = ~60 minutes
- If ingestion approaches 50 minutes → potential failure
- Logged in `failure-log/future_scale_failures.md`

#### Auth System Architecture (3 layers)

1. **Core**
   - `refresh_entity_token(...)`
   - Input: `AuthCredentials`, `TokenState`
   - Output: new `TokenState`

2. **Batch**
   - `batch_refresh(...)`
   - Iterates entities, optional write

3. **Resolution Layer**
   - `resolve_entity_auth_inputs(...)`
   - Combines:
     - `TokenState`
     - `AuthCredentials`
     - `WorkspaceAuthConfig`

#### Contracts Defined
- `AuthCredentials` → raw OAuth credentials
- `TokenState` → persisted token state (per entity)
- `WorkspaceAuthConfig` → workspace → entities mapping

#### Storage Structure
- Tokens:
    ```json
    {
      "entity_name": { ...TokenState }
    }
    ```
- Secrets:
    ```json
    {
    "workspace_name": { ...AuthCredentials }
    }
    ```
