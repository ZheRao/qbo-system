QBO Time Token Renew Procedure
    
    1. open token.json for QBO Time
    2. for each location, open the "URL", double check the "Company Name"
    3. on the left panel, click "Feature Add-ons" -> "API"
    4. double check "OAuth Client Secret" and "OAuth Client ID" and "Token"
    5. extend the expiration date


QBO Token Renew Procedure
    
    1. open client_secrets.json for QBO, use the "redirect_url" 

    2. open tokens.json for QBO for editing

    3. on the website, select workspace and app, double check client_id and client_secret
        - Canada: Sample Workspace + Canada Data (Production)
        - US: Sample Workspace + US Data (Prouction)

    4. under "OAuth settings", select "com.intuit.quickbooks.accounting", then click "Get authorized code"

    5. page will be redirected to select individual locations, select each location, then repeat the following steps
        - Canada: Monette Farms BC Ltd., Monette Farms Ltd., Monette Produce Ltd., Monette Seeds Ltd., Nexgen Seeds Ltd.
        - US: Monette Farms Arizona LLC, Monette Farms USA, Inc., Monette Produce, LLC, Monette Seeds USA, LLC

    repeating following steps for each location
        
        a. after selecting the location, then redirect back to the previous page, check Realm ID with tokens.json file before editing
        b. click "Get tokens"
        c. delete the "refresh_token" from tokens.json
        d. replace the corresponding value with the token from website

        note: you only need to change refresh_token because only refresh_token will be used to refresh auth when perform data extraction
              you can also refresh access_token if it makes you feel better
        

Perform a data extraction process immediately to ensure the tokens are working properly
